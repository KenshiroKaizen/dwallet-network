# The `@pera-io/pera/utils` package

This package contains some utilities that simplify common operations when working with the Pera
TypeScript SDK.

## Constants

A set of constants exported for common uses cases:

- `NPERA_PER_PERA`: The conversion rate for NPERA to PERA (1,000,000,000)
- `PERA_DECIMALS`: the number of decimals you must shift a NPERA value to convert it to PERA (`9`)
- `PERA_ADDRESS_LENGTH`: The number of bytes in a Pera address (32)
- `MOVE_STDLIB_ADDRESS`: The address for the Pera Move standard library
- `PERA_FRAMEWORK_ADDRESS`: The address for the Pera Framework
- `PERA_SYSTEM_ADDRESS`: The address for the Pera System module
- `PERA_CLOCK_OBJECT_ID`: The address for the `pera::clock::Clock` object
- `PERA_SYSTEM_STATE_OBJECT_ID`: The address for the `PeraSystemState` object

## Formatters

You can use the following helpers to format various values:

- `formatAddress`
- `formatDigest`
- `normalizeStructTag`
- `normalizePeraAddress`
- `normalizePeraObjectId`
- `normalizePeraNSName`
- `normalizePeraNSName`

## Validators

You can use the following helpers to validate the format of various values (this only validates that
the value is in the correct format, but does not validate the value is valid for a specific use
case, or exists on chain).

- `isValidPeraAddress`
- `isValidPeraObjectId`
- `isValidTransactionDigest`
- `isValidPeraNSName`

## Encoding

The following methods are re-exported to help with converting between commonly used encodings

- `fromHEX`: Deserializes a hex string to a Uint8Array
- `toHEX`: Serializes a Uint8Array to a hex string
- `fromB64`: Deserializes a base64 string to a Uint8Array
- `toB64`: Serializes a Uint8Array to a base64 string
