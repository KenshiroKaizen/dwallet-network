# PeraClientProvider

The `PeraClientProvider` manages the active `PeraClient` that hooks and components use in the dApp
Kit.

## Usage

Place the `PeraClientProvider` at the root of your app and wrap all components that use the dApp Kit
hooks.

`PeraClientProvider` accepts a list of network configurations to create `PeraClient` instances for the
currently active network.

```tsx
import { createNetworkConfig, PeraClientProvider, WalletProvider } from '@mysten/dapp-kit';
import { getFullnodeUrl } from '@pera-io/pera/client';

// Config options for the networks you want to connect to
const { networkConfig } = createNetworkConfig({
	localnet: { url: getFullnodeUrl('localnet') },
	mainnet: { url: getFullnodeUrl('mainnet') },
});

function App() {
	return (
		<PeraClientProvider networks={networkConfig} defaultNetwork="localnet">
			<YourApp />
		</PeraClientProvider>
	);
}
```

## Props

- `networks`: A map of networks you can use. The keys are the network names, and the values can be
  either a configuration object (`PeraClientOptions`) or a `PeraClient` instance.
- `defaultNetwork`: The name of the network to use by default when using the `PeraClientProvider` as
  an uncontrolled component.
- `network`: The name of the network to use when using the `PeraClientProvider` as a controlled
  component.
- `onNetworkChange`: A callback when the active network changes.
- `createClient`: A callback when a new `PeraClient` is created (for example, when the active network
  changes). It receives the network name and configuration object as arguments, returning a
  `PeraClient` instance.

## Controlled component

The following example demonstrates a `PeraClientProvider` used as a controlled component.

```tsx
import { createNetworkConfig, PeraClientProvider } from '@mysten/dapp-kit';
import { getFullnodeUrl } from '@pera-io/pera/client';
import { useState } from 'react';

// Config options for the networks you want to connect to
const { networkConfig } = createNetworkConfig({
	localnet: { url: getFullnodeUrl('localnet') },
	mainnet: { url: getFullnodeUrl('mainnet') },
});

function App() {
	const [activeNetwork, setActiveNetwork] = useState('localnet' as keyof typeof networks);

	return (
		<PeraClientProvider
			networks={networkConfig}
			network={activeNetwork}
			onNetworkChange={(network) => {
				setActiveNetwork(network);
			}}
		>
			<YourApp />
		</PeraClientProvider>
	);
}
```

## PeraClient customization

The following example demonstrates how to create a custom `PeraClient`.

```tsx
import { PeraClientProvider } from '@mysten/dapp-kit';
import { getFullnodeUrl, PeraClient, PeraHTTPTransport } from '@pera-io/pera/client';

// Config options for the networks you want to connect to
const networks = {
	localnet: { url: getFullnodeUrl('localnet') },
	mainnet: { url: getFullnodeUrl('mainnet') },
} satisfies Record<string, PeraClientOptions>;

function App() {
	return (
		<PeraClientProvider
			networks={networks}
			defaultNetwork="localnet"
			createClient={(network, config) => {
				return new PeraClient({
					transport: new PeraHTTPTransport({
						url: 'https://api.safecoin.org',
						rpc: {
							headers: {
								Authorization: 'xyz',
							},
						},
					}),
				});
			}}
		>
			<YourApp />
		</PeraClientProvider>
	);
}
```

## Using the PeraClient from the provider

To use the `PeraClient` from the provider, import the `usePeraClient` function from the
`@mysten/dapp-kit` module.

```tsx
import { usePeraClient } from '@mysten/dapp-kit';

function MyComponent() {
	const client = usePeraClient();

	// use the client
}
```

## Creating a network selector

The dApp Kit doesn't provide its own network switcher, but you can use the `usePeraClientContext`
hook to get the list of networks and set the active one:

```tsx
function NetworkSelector() {
	const ctx = usePeraClientContext();

	return (
		<div>
			{Object.keys(ctx.networks).map((network) => (
				<button key={network} onClick={() => ctx.selectNetwork(network)}>
					{`select ${network}`}
				</button>
			))}
		</div>
	);
}
```

## Using network specific configuration

If your dApp runs on multiple networks, the IDs for packages and other configurations might change,
depending on which network you're using. You can use `createNetworkConfig` to create per-network
configurations that your components can access.

The `createNetworkConfig` function returns the provided configuration, along with hooks you can use
to get the variables defined in your configuration.

- `useNetworkConfig` returns the full network configuration object
- `useNetworkVariables` returns the full variables object from the network configuration
- `useNetworkVariable` returns a specific variable from the network configuration

```tsx
import { createNetworkConfig, PeraClientProvider } from '@mysten/dapp-kit';

import { createNetworkConfig, PeraClientProvider, WalletProvider } from '@mysten/dapp-kit';
import { getFullnodeUrl } from '@pera-io/pera/client';
import { QueryClient, QueryClientProvider } from '@tanstack/react-query';

// Config options for the networks you want to connect to
const { networkConfig, useNetworkVariable } = createNetworkConfig({
	localnet: {
		url: getFullnodeUrl('localnet'),
		variables: {
			myMovePackageId: '0x123',
		}
	},
	mainnet: {
		url: getFullnodeUrl('mainnet'),
		variables: {
			myMovePackageId: '0x456',
		}
	},
});

const queryClient = new QueryClient();

function App() {
	return (
		<QueryClientProvider client={queryClient}>
			<PeraClientProvider networks={networkConfig} defaultNetwork="localnet">
				<WalletProvider>
					<YourApp />
				</WalletProvider>
			</PeraClientProvider>
		</QueryClientProvider>
	);
}


function YourApp() {
	const id = useNetworkVariable('myMovePackageId');

	return <div>Package ID: {id}</div>;
}
```
