# Rpc Hooks

Ika dApp Kit ships with hooks for each of the RPC methods defined in the
[JSON RPC specification](https://docs.ika.io/ika-api-ref).

## useIkaClientQuery

Load data from the Ika RPC using the `useIkaClientQuery` hook. This hook is a wrapper around the
[useQuery](https://tanstack.com/query/latest/docs/react/guides/queries) hook from
@tanstack/react-query.

The hook takes the RPC method name as the first argument and any parameters as the second argument.
You can pass any additional `useQuery` options as the third argument. You can read the
[useQuery documentation](https://tanstack.com/query/latest/docs/react/guides/queries) for more
details on the full set of options available.

```tsx
import { useIkaClientQuery } from '@mysten/dapp-kit';

function MyComponent() {
	const { data, isPending, isError, error, refetch } = useIkaClientQuery(
		'getOwnedObjects',
		{ owner: '0x123' },
		{
			gcTime: 10000,
		},
	);

	if (isPending) {
		return <div>Loading...</div>;
	}

	if (isError) {
		return <div>Error: {error.message}</div>;
	}

	return <pre>{JSON.stringify(data, null, 2)}</pre>;
}
```

## useIkaClientQueries

You can fetch a variable number of Ika RPC queries using the `useIkaClientQueries` hook. This hook
is a wrapper around the
[useQueries](https://tanstack.com/query/latest/docs/react/reference/useQueries) hook from
@tanstack/react-query.

The `queries` value is an array of query option objects identical to the `useIkaClientQuery` hook.

The `combine` parameter is optional. Use this parameter to combine the results of the queries into a
single value. The result is structurally shared to be as referentially stable as possible.

```tsx
import { useIkaClientQueries } from '@mysten/dapp-kit';

function MyComponent() {
	const { data, isPending, isError } = useIkaClientQueries({
		queries: [
			{
				method: 'getAllBalances',
				params: {
					owner: '0x123',
				},
			},
			{
				method: 'queryTransactionBlocks',
				params: {
					filter: {
						FromAddress: '0x123',
					},
				},
			},
		],
		combine: (result) => {
			return {
				data: result.map((res) => res.data),
				isSuccess: result.every((res) => res.isSuccess),
				isPending: result.some((res) => res.isPending),
				isError: result.some((res) => res.isError),
			};
		},
	});

	if (isPending) {
		return <div>Loading...</div>;
	}

	if (isError) {
		return <div>Fetching Error</div>;
	}

	return <pre>{JSON.stringify(data, null, 2)}</pre>;
}
```

## useIkaClientInfiniteQuery

For RPC methods that support pagination, dApp Kit also implements a `useIkaClientInfiniteQuery`
hook. For more details check out out the
[`useInfiniteQuery` documentation](https://tanstack.com/query/latest/docs/react/guides/infinite-queries).

```tsx
import { useIkaClientInfiniteQuery } from '@mysten/dapp-kit';

function MyComponent() {
	const { data, isPending, isError, error, isFetching, fetchNextPage, hasNextPage } =
		useIkaClientInfiniteQuery('getOwnedObjects', {
			owner: '0x123',
		});

	if (isPending) {
		return <div>Loading...</div>;
	}

	if (isError) {
		return <div>Error: {error.message}</div>;
	}

	return <pre>{JSON.stringify(data, null, 2)}</pre>;
}
```

## useIkaClientMutation

For RPC methods that mutate state, dApp Kit implements a `useIkaClientMutation` hook. Use this hook
with any RPC method to imperatively call the RPC method. For more details, check out the
[`useMutation` documentation](https://tanstack.com/query/latest/docs/react/guides/mutations).

```tsx
import { useIkaClientMutation } from '@mysten/dapp-kit';

function MyComponent() {
	const { mutate } = useIkaClientMutation('dryRunTransactionBlock');

	return (
		<Button
			onClick={() => {
				mutate({
					transactionBlock: tx,
				});
			}}
		>
			Dry run transaction
		</Button>
	);
}
```

## useResolveIkaNSName

To get the IkaNS name for a given address, use the `useResolveIkaNSName` hook.

```tsx
import { useResolveIkaNSName } from '@mysten/dapp-kit';

function MyComponent() {
	const { data, isPending } = useResolveIkaNSName('0x123');

	if (isPending) {
		return <div>Loading...</div>;
	}

	if (data) {
		return <div>Domain name is: {data}</div>;
	}

	return <div>Domain name not found</div>;
}
```
